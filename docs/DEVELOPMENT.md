## Local Setup

### 1. Clone & Configure

```bash
git clone <repo-url>
cd healthconnect
```

Create `backend/.env` with the following:

```env
DATABASE_URL=postgresql://user:pass@host:5432/dbname
SUPABASE_URL=https://<project-id>.supabase.co
SUPABASE_KEY=<anon-public-key>
SUPABASE_SERVICE_ROLE_KEY=<service-role-secret-key>
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your@email.com
SMTP_PASS=your-app-password
FROM_EMAIL=noreply@healthconnect.com
```

> ⚠️ `SUPABASE_SERVICE_ROLE_KEY` is a **secret**. Never expose it in the frontend or commit it to version control.

---

### 2. Backend

```bash
cd backend
python -m venv venv
venv\Scripts\activate      # Windows
pip install -r requirements.txt
alembic upgrade head        # Apply DB migrations
uvicorn main:app --reload   # Start dev server → http://localhost:8000
```

**API Docs** (auto-generated by FastAPI):
- Swagger UI: http://localhost:8000/docs
- Redoc: http://localhost:8000/redoc

---

### 3. Frontend

```bash
cd frontend
npm install
npm run dev                 # Start dev server → http://localhost:5173
```

---

## Running Tests

```bash
cd backend
pytest
```

Tests run against a local SQLite `test.db`. They do **not** require Supabase credentials.

## Common Issues & Solutions

### Backend won't start
- Ensure `venv` is activated.
- Check `DATABASE_URL` is a valid PostgreSQL URI (not SQLite for production).
- Run `alembic upgrade head` to ensure migrations are applied.

### Frontend API calls return 422
- This is a FastAPI validation error. Check the `errors` field in the response body.
- Ensure all required body/query params match the API schema.

### Frontend crashes on page load after backend update
- If a list endpoint was recently paginated, the frontend may be calling `.map()` on an object instead of an array.
- Ensure the component reads `res.data.items || res.data` instead of `res.data` directly.

### OTP email not arriving
- Verify SMTP credentials in `.env`.
- Check spam folder.
- OTPs expire after 10 minutes — request a new one if needed.

### Admin "Add User" fails
- Ensure `SUPABASE_SERVICE_ROLE_KEY` is set in `backend/.env`.
- The anon key does not have permission to create users server-side.
